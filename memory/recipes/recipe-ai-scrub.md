# AI-Reference Scrub Recipe

Rewrites git history to remove AI-attribution signals from a repo before public scrutiny. Addresses the AI-stigma concern: recruiters, collaborators, and prospective employers scanning commit history shouldn't see "Co-Authored-By: Claude" or "Generated with Claude Code" on every commit.

## What gets scrubbed

### 1. Commit messages

Patterns removed from the message body:

- `Co-Authored-By: Claude <noreply@anthropic.com>` (any email variant)
- `Co-Authored-By: Claude Code <...>`
- `Co-Authored-By: Codex <...>` / `Co-Authored-By: OpenAI Codex`
- `Co-Authored-By: GitHub Copilot <...>`
- `Generated with [Claude Code](https://...)` (markdown links)
- `🤖 Generated with Claude Code` (and emoji variants: 🤖, 🦾)
- `Via Claude Code`, `via Copilot`, `via ChatGPT`
- URLs to `claude.ai`, `anthropic.com/claude-code`, `chat.openai.com`, `chatgpt.com`
- Standalone lines mentioning: "AI assistant", "written by Claude", "assisted by AI", "auto-generated by", "Cursor", "Cline", "Aider" (when referring to AI authorship, not the person)
- Trailing signature blocks added by AI tools

### 2. Commit authors (optional, opt-in)

If the `author.name` or `author.email` identifies an AI tool (e.g., `claude@anthropic.com`, `codex@openai.com`), rewrite to the user's real identity.

### 3. Tracked files

Purge from all history (not just current tree):

- `CLAUDE.md`, `CODEX_CHANGELOG.md`, `CLAUDE_NOTES.md`
- `.claude/` directory and contents
- `.codex/` directory and contents
- `AGENTS.md`, `AI_CONTEXT.md`, `.aider*` files
- `.cursorrules`, `.continuerc`, `.clinerc`
- Any file the user flags specifically

### 4. What does NOT get scrubbed (intentionally)

- Commit SHAs, dates, diffs, parentage — only the message + metadata change unless files are purged.
- Legitimate uses of words like "claude" or "codex" that aren't AI attribution (e.g., a repo actually named "claude" or a variable called `codex_`). Patterns are anchored to trailer/signature shape to avoid false positives.
- GitHub PR / issue comments — those live on GitHub's servers, not in the repo. Scrubbing requires manually editing or deleting comments via the GitHub API or UI.

## Invoke

Trigger phrases:
- **"scrub AI references from &lt;repo&gt;"**
- **"clean AI commits in &lt;repo&gt;"**
- **"run the AI scrub on &lt;repo&gt;"**

Copy the prompt from `~/repos/ai-prompts/ai-scrub-prompts.txt` and paste.

Or run the helper script directly:
```bash
~/.claude-octopus/bin/ai-scrub.sh <repo-path> [--dry-run|--apply] [--include-files] [--include-authors]
```

## Directive

### Phase 1 — Precondition checks

1. **Verify repo is clean.** `rtk git status --porcelain` must be empty. Stash WIP first if needed.
2. **Verify `git-filter-repo` is installed.** If not:
   ```bash
   pip install --user git-filter-repo
   ```
3. **Record current state:**
   - Current branch name.
   - Current HEAD SHA.
   - Remote list.
4. **Shared-repo check.** If the repo has other contributors or is widely cloned, warn that history rewrite will invalidate other clones. User must explicitly confirm before proceeding.

### Phase 2 — Backup (non-optional)

Two-layer backup before any rewrite:

1. **Local bundle:**
   ```bash
   rtk git bundle create ~/repos/backups/<repo-name>-<timestamp>.bundle --all
   ```
2. **Backup branch on remote:**
   ```bash
   rtk git push origin <current-branch>:refs/heads/pre-ai-scrub-<timestamp>
   ```

Both must succeed before Phase 3 runs. On failure, halt.

### Phase 3 — Dry-run

Always runs first. Executes the full scrub logic but writes results to a throwaway clone (`/tmp/ai-scrub-<timestamp>/`), not the real repo. Produces a report:

- Count of commits whose messages will change
- Count of Co-Authored-By trailers removed
- Count of files purged from history (if `--include-files`)
- Count of author-identity rewrites (if `--include-authors`)
- Sample of 5 before/after message pairs
- List of any commits that will become empty (message + diff both scrubbed to nothing) — these should be preserved as empty-message commits, not dropped.

User reviews the report. Dry-run NEVER modifies the real repo.

### Phase 4 — Apply (explicit opt-in only)

Only runs if user passes `--apply` (script) or explicitly confirms (agent mode).

```bash
cd <repo>
git filter-repo \
  --message-callback '<scrub-callback>' \
  [--invert-paths --path CLAUDE.md --path CODEX_CHANGELOG.md --path .claude/ ...] \
  --force
```

After filter-repo runs:

1. Verify HEAD moved (SHAs should differ).
2. Spot-check 3 random commits: `rtk git log --format=fuller -1 <sha>` — messages should be scrubbed.
3. Run secret scan across new history (`directive-secret-scan.md` patterns against all commits) — any leaked secret that survived deserves extraction now.
4. **filter-repo strips remotes by default.** Re-add origin:
   ```bash
   rtk git remote add origin <url>
   ```

### Phase 5 — Force-push (explicit confirmation)

Rewriting history REQUIRES force-push. Warn one more time:

```
⚠ Force-pushing rewrites remote history. All existing clones become invalid
  and must be re-cloned. Existing PRs may show phantom diffs.
  Backup exists at: pre-ai-scrub-<timestamp> branch on origin.
  Backup bundle at: ~/repos/backups/<repo-name>-<timestamp>.bundle.
  Continue? [yes/no]
```

On yes:

```bash
rtk git push --force-with-lease origin <branch>
rtk git push --force-with-lease origin --tags
```

`--force-with-lease` (not `--force`) — aborts if someone else pushed in the meantime.

### Phase 6 — Verify remote

1. `rtk curl` a known-scrubbed commit URL with cache-bust — confirm GitHub serves the new message.
2. Check GitHub's commit list in the web UI — spot-check 3 recent messages.
3. Report to user: number of commits rewritten, backup locations, any commits that couldn't be cleanly scrubbed (with reason).

### Phase 7 — Cleanup

1. Delete the temp clone (`/tmp/ai-scrub-<timestamp>/`).
2. Keep backup bundle for 90 days (user deletes manually when confident).
3. Keep remote backup branch for 30 days, then prompt user to delete.

## Non-Negotiable Rules

- **Never scrub without backup.** Phase 2 is mandatory and must succeed before Phase 3.
- **Never skip dry-run.** `--apply` cannot run without a preceding dry-run in the same session.
- **Never force-push without explicit confirmation.** Even with `--apply`, push is a separate explicit step.
- **`--force-with-lease`, never `--force`.** Catches concurrent pushes.
- **Respect shared-repo warning.** If the repo has other contributors, the agent surfaces the warning and requires explicit ack before proceeding.
- **Preserve legitimate content.** Never remove code, never strip commit bodies that discuss the AI-scrub itself (meta-commits are OK to keep).
- **Handle empty commits.** If scrubbing removes the entire message, keep the commit with a placeholder message (`"(message removed)"`). Do NOT drop the commit — SHAs downstream depend on it.
- **Do not touch signed commits.** If any commit is GPG-signed, warn user — rewrite invalidates signatures. User must opt-in to proceed.

## Patterns library (reference for the scrub callback)

Full regex set maintained in the helper script. These run against the full commit message (subject + body):

```
# Trailers
Co-Authored-By:\s*Claude(\s+Code)?\s*<[^>]*>\s*
Co-Authored-By:\s*(OpenAI\s+)?Codex\s*<[^>]*>\s*
Co-Authored-By:\s*GitHub\s+Copilot\s*<[^>]*>\s*

# Generated-with signatures
(^|\n)\s*(🤖|🦾)?\s*Generated with \[?Claude Code\]?(\([^)]*\))?\s*$
(^|\n)\s*Generated by Claude(\s+Code)?(\s+v?\d+(\.\d+)*)?\s*$
(^|\n)\s*Made with Claude(\s+Code)?\s*$

# URL-based attributions
https?://claude\.ai\S*
https?://(www\.)?anthropic\.com/claude-code\S*
https?://(chat\.)?openai\.com\S*
https?://chatgpt\.com\S*

# Via-phrases (anchored to end of line to avoid false positives)
\n\s*(via|using)\s+(Claude(\s+Code)?|Codex|ChatGPT|Copilot|Cursor|Cline|Aider)\s*$

# Blank-line cleanup (multi-newline sequences left after removals)
\n{3,}  →  \n\n
\s+$    →  (trailing whitespace stripped)
```

False-positive safeguards:

- Patterns are anchored to trailer shape, URL shape, or line boundaries — prevents matching legitimate prose like "we used to handle codex files" or "the claude release".
- The scrub callback preserves the subject line (first line) unless the ENTIRE subject matches a pattern (rare; usually the subject is the legit commit summary).
- If a commit body becomes empty after scrubbing, the subject stands alone.

## Gotchas

- **GitHub Actions / CI caches** — after force-push, clear any action cache that includes the old commit SHAs (e.g., `gh cache delete --all`).
- **Submodules** — if the repo has submodules, scrubbing the parent doesn't affect submodule history. Run scrub on each submodule separately.
- **Git LFS** — filter-repo handles LFS pointers correctly, but run `git lfs fsck` after to confirm.
- **Tags** — annotated tag messages can also contain AI references. The helper script includes `--replace-message` for tag messages.
- **PRs with the old commits** — GitHub pulls may show a "this branch is N commits ahead/behind" badge incorrectly. Close and re-open the PR to resync, or force-push to the PR branch as well.
