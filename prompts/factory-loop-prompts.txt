 FACTORY LOOP — One prompt, zero fill-in, works on any repo
=============================================================================
 How to use:
   1. Tell Claude: "Pull up ~/repos/<PROJECT>"   (or paste the full path)
   2. Copy the prompt below (between the ===== lines)
   3. Paste. Send. Walk away.

 The prompt auto-detects: new-vs-existing, stack, current version, themes,
 scope guards, iteration count, and release target. Nothing to fill in.

 CANONICAL ROUTING (default) — copilot-heavy:
   The default preset is now copilot-heavy so Claude Max quota is preserved
   for genuine escalation only. Bulk research, synthesis, implementation,
   counter-passes, UX, theming, and audit all route through Copilot's
   Sonnet 4.6 and GPT-5.3-Codex. Gemini Flash seeds broad research. Direct
   Claude Max fires only when the work genuinely needs Opus-level reasoning
   (PEC rubric UNCERTAIN ≥3 rounds, debate stalemate, security escalation).

   Swap routing: ~/.claude-octopus/bin/octo-route.sh <mode>
   Modes: copilot-heavy (default) | balanced | claude-heavy | codex-heavy |
          direct-only | copilot-only

 Design principle of this prompt:
   The RECIPE is the source of truth. This prompt does not restate recipe
   internals — doing that is what caused drift in earlier versions. Updates
   to the flow (new phases, new gates, new directives) belong in
   recipe-factory-loop.md, not here. This prompt handles only the dynamic
   bits (path, goal, iteration count), the offload policy, the research
   expectation, and a minimal hard-cap/never backstop.

  first-pass AND counter-pass, audit first-pass. The master Claude session
  (this session) dispatches, coordinates, and reviews — it does NOT handle

    1. A PEC rubric returned UNCERTAIN on ≥3 rounds on the same task (Opus

    2. A three-role debate stalemates (KS distance never drops below 0.05
       across max rounds). OR

       where Sonnet 4.6 via Copilot would be undershooting.

  doing bulk implementation in this Claude session without an escalation
  reason is a routing bug — stop and delegate.

RESEARCH EXPECTATION (default is "extensive", not "quick"):
- L1a research runs ALL 9 dimensions on iter 1, regardless of whether
  ROADMAP.md already has items. Existing ROADMAP items do NOT suppress
  research — research expands the ROADMAP with high-leverage tasks the
  existing list missed.
- The 9 research dimensions (see recipe-factory-loop.md L1a for full detail):
    1. Competitor feature parity (top 3 OSS + top 3 commercial peers)
    2. Recent upstream releases (dependency changelogs, 60-day window)
    3. CVE / security advisories (OSV.dev + GitHub Advisory DB)
    4. Accessibility gaps (WCAG 2.2 AA dimensions)
    5. Performance regressions (bundle/startup/memory drift)
    6. UX / GUI polish opportunities (interactive-state coverage, motion,
       microcopy, error-message specificity)
    7. Theme coverage (dark-mode edge cases, interactive-state coverage
       across all themes)
    8. Community asks (last 90 days of issues/discussions/PRs — this repo
       AND top 2 peers)
    9. Platform / ecosystem shifts (deprecations on the clock)
- Research output lands in `docs/research/iter-<N>-landscape.md`, one
  section per dimension, marked UNTRUSTED DATA. Each ROADMAP task added by
  L1b MUST cite which research dimension it came from.
- If the project is a GUI app, the UX / theme / accessibility dimensions
  get weighted higher in task ranking.
- Iteration 2+ does DELTA research only — net-new findings since last iter.

Lazy-load directives only when the phase that needs them fires:

AUTO-DETECT:
- No .git directory → NEW project. Run full preflight (scaffold + OSS research
  + logo + gh repo create --private + branch protection). S-phase auto-skips
  (no prior history to scrub). W-phase auto-skips (no WIP yet).
- Has .git → EXISTING project. Pass --skip-preflight. W-phase auto-runs if
  uncommitted work present. S-phase auto-runs early to scrub AI-reference
  history (autopilot: backup + apply + push; downgrades to manual on
  multi-contributor or GPG-signed commits). G-phase auto-runs if icon set
  missing (SVG-via-Copilot primary, no OpenAI billing needed).
- Pick the best stack for the problem on NEW projects (per "Best tool for the
  job" rule in ~/.claude/CLAUDE.md — do not default to Python or PowerShell).
  Load the matching stack-*.md after the decision.
- On EXISTING projects: detect stack from build files (package.json / Cargo.toml
  / *.csproj / build.gradle / go.mod / pyproject.toml / etc.). Load the matching
  stack-*.md.
- Read repo CLAUDE.md + linked memory file + ROADMAP.md + rtk git log -20 before
  deciding the goal on EXISTING projects.
- Derive the goal from repo state AFTER running L1 research (research may
  promote a different top priority than the current ROADMAP suggests):
    ROADMAP has unchecked items + research surfaces high-impact additions
      → advance ROADMAP with research additions prioritized
    Pending release → ship it AFTER L3/L4 audit closes cleanly
    Open audit findings → close them, then replenish + advance
    Empty / clean → hardening + polish pass, still run full L1 research to
      seed the next ROADMAP round
- Derive iteration count (higher than prior defaults — default is heavy):
    NEW = 5 iterations
    EXISTING with active roadmap = 4 iterations
    EXISTING clean (no roadmap, no findings) = 2 iterations (research + one
      implementation cycle minimum — don't ship "nothing changed" runs)
  Stop-on-convergence and stop-on-regression fire regardless of the count.
- Next version: current patch bump unless roadmap work is major/breaking (minor
  bump). Detect current via rtk grep on manifest/README badge. The single bump
  happens at Q3, NOT per iteration.
- Scope guards: extract from repo CLAUDE.md + memory file charter language.
  Reject ROADMAP items that violate the stated charter (tag as CHARTER-REVIEW
  and defer — do NOT silently drop, the user may want to expand charter).
